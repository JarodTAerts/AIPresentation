the tools

What "AI coding tools" actually are

Three generations, all live in market today.

Gen 1
Autocomplete
GitHub Copilot, Cursor Tab. Suggests the next 1–10 lines as you type. Lives inside the editor.
Gen 2
Chat & edit
Copilot Chat, Cursor Composer. "Refactor this function." "Add a test for X." Reads files, proposes diffs, you apply.
Gen 3
Agents
Claude Code, Copilot CLI, Devin, Aider. Run shell commands, edit many files, run tests in a loop until the task is done — minutes of autonomous work per turn.

How they actually work

  1. Gather context — read the open file, related files, project structure.
  2. Send to an LLM with a system prompt that says "you are a coding assistant, here are your tools."
  3. Stream the response back. If it's an agent, execute any tool calls (read_file, run_tests, edit) and feed results back.
  4. Loop until the model says "done" or the user stops it.

The numbers

  • 84% of developers use AI tools daily/weekly (Stack Overflow 2025).
  • 55% faster on representative coding tasks in GitHub's own studies.
  • BUT: METR's RCT with experienced open-source devs found AI made them 19% slower on familiar code — they overestimated speedup by 20pp.

Sources: GitHub Octoverse 2025, METR Measuring the Impact of Early-2025 AI on Experienced Open-Source Developer Productivity.
